================================================================
  STARTING FRESH WITH A NEW EXCEL FILE
  Read this entire section before touching anything else.
================================================================

  STEP A — PREPARE THE EXCEL FILE
  --------------------------------
  The Excel file MUST be formatted exactly like this:
    - Sheet name must be:  Sheet1   (exactly, capital S)
    - No header row skip needed — row 1 is skipped automatically
    - Must have exactly 10 columns in this order:

      Col A  |  ISBN
      Col B  |  Book Title / Name
      Col C  |  Author
      Col D  |  Publisher
      Col E  |  Binding Type  (e.g. Paperback, Hardcover)
      Col F  |  Language
      Col G  |  (any column — ignored by the script)
      Col H  |  Price  (number)
      Col I  |  Stock / Quantity  (number)
      Col J  |  (any column — ignored by the script)

  If the columns are in a different order in the new file,
  you MUST update enrich_books.py (see Step B below).


  STEP B — UPDATE THE FILENAME IN enrich_books.py
  -------------------------------------------------
  Open:  enrich_books.py
  Find line 20 (near the top, under "# ── Config"):

      EXCEL_FILE = "common distributor stock IBD PBI 10.06.2026.xlsx"

  Change the filename to match your new Excel file exactly:

      EXCEL_FILE = "your new excel file name here.xlsx"

  The Excel file must be in the SAME folder as the scripts.

  Optional — also change CHUNK_SIZE on line 27 if needed:
      CHUNK_SIZE = 2000   <- 2000 books per output CSV (recommended)
                             0 = put everything in one file


  STEP C — CLEAR OLD RUN STATE
  -----------------------------
  Before running on a new Excel, delete or rename these files
  so the script starts fresh:

      Delete:  progress.json            (tracks last processed row)
      Delete:  shopify_import_*.csv     (old output files)
      Delete:  rejected_no_image.csv    (old rejected books log)

  KEEP (optional but recommended):
      Keep:    isbn_cache.json          (56,000+ already-looked-up ISBNs)
                                         Keeping it saves API calls for
                                         books that appeared in old run.

  To delete the old CSVs quickly, open terminal in the project
  folder and run:
      del shopify_import_*.csv
      del rejected_no_image.csv
      del progress.json

  Then double-check nothing is left:
      dir shopify_import_*.csv


  STEP D — RUN THE MAIN SCRIPT
  -----------------------------
  Once Steps A, B, C are done:

      py run.py

  This will process your entire new Excel file from row 1.

================================================================
  BEFORE UPLOADING TO SHOPIFY — VERIFY IMAGES
================================================================

    py verify_images.py shopify_import_001.csv

  Run this on any CSV before uploading to Shopify.
  It checks every image URL is real (live HTTP check).

  Result codes:
    OK    = real image confirmed (HTTP 200, correct type, size > 2KB)
    FAIL  = bad URL / wrong HTTP status — needs fixing
    TINY  = file under 2KB — likely a placeholder image
    ERR   = connection timeout — retry or replace manually
    WARN  = no Content-Type header — usually still fine

  Run on any file by changing the filename:
    py verify_images.py shopify_import_002.csv
    py verify_images.py shopify_import_003.csv

  Target: 0 FAIL, 0 TINY, 0 ERR before uploading.


================================================================
  CHECK A CSV's STATS (images + descriptions coverage)
================================================================

  Paste this into terminal (change the filename on line 5):

    py -c "
    import csv, re
    CSV = 'shopify_import_001.csv'
    def ri(u): return bool(u) and 'covers.openlibrary.org/b/isbn/' not in u and 'covers.openlibrary.org/b/id/-1-' not in u
    def rd(h): return bool(re.search(r'<p>.+?</p>', h, re.DOTALL))
    t=ri_=rd_=b=n=ni=0
    with open(CSV, encoding='utf-8-sig') as f:
      for row in csv.DictReader(f):
        if not row.get('Variant SKU','').strip(): continue
        t+=1; i=ri(row.get('Image Src','')); d=rd(row.get('Body (HTML)',''))
        if i: ri_+=1
        if d: rd_+=1
        if i and d: b+=1
        if not i and not d: n+=1
        if not i: ni+=1
    print('Total:',t,'| Images:',ri_,'| Desc:',rd_,'| Both:',b,'| Neither:',n,'| No image:',ni)
    "

================================================================
  TROUBLESHOOTING
================================================================

  Script seems frozen on one book:
    Press Ctrl+C and re-run. It resumes from last saved row.
    progress.json tracks the exact row it was on.

  All crosses (x) shown in terminal:
    Normal — crosses mean ISBNdb had no data for that book.
    Green OK = found image or description.
    The daily limit hasn't been wasted — crosses just skip fast.

  "0 patches" after --rewrite-only:
    Expected. CSVs were already updated during the last run.
    Rewrite runs automatically when daily limit is hit.

  429 rate limit hit too early:
    You've used all 5,000 daily calls. Wait until tomorrow,
    or upgrade to a paid ISBNdb plan.

  Image shows FAIL in verify_images.py:
    The URL is broken. Find a replacement on Goodreads:
      https://www.goodreads.com/book/isbn/XXXXXXXXXX
    Then manually update isbn_cache.json for that ISBN
    and run:  py patch_descriptions.py --rewrite-only

  verify_images.py shows TINY (< 2KB):
    Likely a placeholder. Treat same as FAIL — find a
    real image from Goodreads and patch it.

  New Excel has different column order:
    Open enrich_books.py, find line ~133 in run.py:
      (_, name, author, publisher, binding, language,
       _, price, stock, _) = row
    Reorder the variable names to match your column order.
    Also update EXCEL_FILE in enrich_books.py (line 20).

  "Sheet1 not found" error on run.py:
    Your Excel sheet tab is named something else.
    Open enrich_books.py, find:  wb["Sheet1"]
    Change "Sheet1" to whatever your sheet tab is named.
